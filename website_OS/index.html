Problem Overview

The ProducerConsumer problem is an implementation of two threads producer and consumer where it uses two queues in two different ways accessing these queues.

One of the solutions that I used in this project was where the producer has its own queue but the consumer can consume from both queues. The second solution was where each of the consumers has its own queue and the producer could produce from both queues but chooses the shortest size queue of either queue.

In this project I made a comparison with these two solutions to get the average time for when a producer thread produces an item and the consumer thread waits in the queue these comparison determines which of the two solutions its better to solve the ProducerConsumer problem.

Interesting Implementations & Difficulties

In the project I came up with an algorithm where I can compute the averages of when the producer produces time stamps to the queue and when the consumer waits on the queue this algorithm was implemented in a method where it will be called on the producer and consumer thread. A difficulty that I had encountered was where I had to create an implementation so that the producer or the consumer chooses from either queues.

Results & Analysis

To get the results I needed I compared both solutions by setting test sizes to 2producers/2consumer, and 1producer/1consumer. The results I got from my first solution was that the producers were finishing faster than the consumer threads while in the second solution it was the same as well. But in the second solution the producer and consumer threads were running much faster than the first solution. This says that the second solution where each consumer has its own queue, and the producers can produce in either queue by choosing the shortest one, is the best solution for the problem.
